Components

This page describes the main post-processing components in COMPS. Each component plays a role in transforming or modifying the forecasts from the data source to the final product.

 

Input

Inputs are data sources of weather information. This can be in form of observations or model output (forecasts). Data can be provided in a variety of formats. The easiest is a simple text format, which uses one row for each forecast horizon, and one column for each ensemble member. One file is used for each location and date. COMPS supports weather forecasts in the standardized Netcdf file-format, with CF naming conventions. In addition Grib can also be used.

For observations, COMPS supports METAR data provided in NCAR's NetCDF format.

Inputs can also simulate weather information. For example the Sinusoidal input produces a sinusoidal signal of a set frequency and amplitude. This is useful to test the performance of schemes against expected outcomes. Similarly, Lorenz63 simulates a simple 3-state model of atmospheric convection.

Selection

The selection component constructs an ensemble by selecting among. Typically an ensemble is constructed by selecting forecast from different runs. We have generalized this consept by allowing forecasts—or observations— from different initialization dates and times, datasets, and forecast horizons. In this way, climatology and persistence ensembles can be created as well as lagged-time ensembles.

Downscaling

The forecast datasets used are often defined for different locations that where we want forecasts for. The process of determining forecast values for the desired location is called downscaling.

Correction

The correction component bias-corrects each ensemble member. Forecasts from models often exhibit biases. Typically, a correction is made by estimating the bias when compared to observations. Other

Uncertainty

The uncertainty model is the link between an ensemble and a probability distribution.

The uncertainty component typically produces a continuous distribution, valid for the domain of the forecast variable. For some variables, such as precipication accumulation, a separate discrete model is used for the boundary (0 mm).

Calibration

In the same way that ensemble contain biases, probability distribution can also be biased. The calibration component enables distributional bias to be removed. This can be achieved by ensuring that observations fall evenly among the percentiles in the distribution.

Output

Various products can be produced from the calibrated forecast distributions. Probability density and cumulative probabilities can be computed for any set of thresholds. These values are retrieved by the output and placed in files on the disk. These can be files with human-readable text, in NetCDF, XML, or JSON format. Custom formats can also be created.
