Schemes

The schemes available in comps are described in the Schemes section. These schemes are defined in C++ code and compiled. However, many schemes have options that modify their behaviour. These options are specified in ./namelists/default/schemes.nl. The schemes have been grouped into the components defined in Schemes. As an example, scroll down to the Downscaler section: {% highlight python %} # Downscaler nearest class=DownscalerNearestNeighbour distance class=DownscalerDistance order=2 numPoints=16 {% endhighlight %}

Here two downscaling schemes have been set up. First, the nearest neighbour scheme has been defined, and given the name nearest. Configurations wishing to use this schemes would use this name to refer to it. Below is the declaration of a downscaler that weights nearby grid points.This scheme uses the 16 nearest neighhours, and weights them based on the square of the inverse distance.

The add a scheme, look up the class name in Schemes and fill in any of the options.
