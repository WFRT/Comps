Configurations

Configurations are defined in ./namelists/default/configurations.nl. A configuration is composed of the following post-processing components:

Options

Each configuration has many options that can be specified. The most important ones are: selector, correctors, continuous, and lowerDiscrete (for Precipitation). The defaults for the other other options should at least get you started. {% include ConfigurationsConfiguration.html %} Required: will not run without these
Optional.

Example

As an example, consider the following configuration declaration: {% highlight python %} clim selector=clim downscaler=nearest averager=mean continuous=bpe {% endhighlight %}

Here the clim selection scheme, the nearest downscaler, the mean averager, and the bpe continuous probability model has referenced from ./namelists/default/schemes.nl. Note that the configuration name in this case (clim) is the same as the selector it uses. Names can be reused between different namelist files without causing conflicts. It makes sense to call this configuration clim because it is the selector that best defines this configuraiton. The other schemes (downscaler, averager, and continuous) are standard schemes.
